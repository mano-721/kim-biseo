# /배포

작업 디렉토리 변경 사항을 자동으로 GitHub에 커밋하고 푸시합니다.

## 실행 흐름

### 1단계: 변경된 파일 확인
```bash
git status --short
```
스테이징되지 않은 파일, 수정 사항을 확인합니다.

### 2단계: 커밋 메시지 자동 생성
변경 파일 목록을 분석하여 커밋 메시지를 자동 생성합니다.

**규칙:**
- 파일 생성 → `feat:` 
- 파일 수정 → `update:` 또는 `fix:`
- 문서 수정 → `docs:`
- 여러 파일 → 주요 변경사항 요약

예시:
- `feat: dashboard.html 및 데이터 파일 추가`
- `update: 업무목록.csv 데이터 갱신`
- `docs: 프로젝트 문서 업데이트`

### 3단계: GitHub에 푸시
```bash
git add .
git commit -m "[자동 생성된 메시지]"
git push
```

### 4단계: 배포 완료 메시지
```
✅ 배포 완료!
- 파일 변경: [개수]
- 커밋: [커밋 해시]
- 리포지토리: https://github.com/mano-721/kim-biseo
- Vercel 배포: https://kim-biseo.vercel.app (자동 연동)
```

## 사용 예시

```
/배포
```

변경된 파일이 없으면:
```
ℹ️ 변경사항이 없습니다.
```

## 주의사항

- 비밀번호나 API 키 같은 민감한 정보는 포함하면 안 됩니다.
- `.env` 파일은 `.gitignore`에 추가되어 자동 제외됩니다.
- Vercel과 GitHub이 연동되어 있으므로 푸시 후 자동 배포됩니다.
