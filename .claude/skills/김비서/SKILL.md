# 김비서 — 대시보드 업데이트 & 데일리 브리핑

사용자의 업무 데이터를 읽고, 대시보드를 최신 상태로 갱신하고, 오늘의 브리핑을 제공합니다.

## 작업 경로

- 작업 디렉토리: `/Users/seomanho/Downloads/워크숍-실습/`
- 데이터 폴더: `김비서-데이터/`
- 대시보드: `김비서-데이터/dashboard.html`
- 정리 폴더: `정리해줘/`

## 실행 순서

### 1단계: 데이터 수집

아래 파일들을 **모두** 읽어서 현재 상태를 파악합니다:

| 파일 | 내용 |
|------|------|
| `김비서-데이터/매출데이터.csv` | 제품별 매출, 수량, 지역 데이터 |
| `김비서-데이터/업무목록.csv` | 업무, 우선순위, 상태, 담당자, 마감일 |
| `김비서-데이터/주간일정.txt` | 이번 주 일정 (요일별 미팅, 마감 등) |
| `김비서-데이터/프로젝트현황.csv` | 프로젝트 진행률, 예산, 상태 |
| `김비서-데이터/회의록.txt` | 최근 회의 내용, 액션 아이템 |
| `김비서-데이터/dashboard.html` | 현재 대시보드 HTML |

추가로 `정리해줘/` 폴더의 하위 구조(보고서, 메모, 업무, 기타)도 확인합니다.

### 2단계: 대시보드 업데이트

`dashboard.html`의 각 섹션을 데이터 파일 내용 기준으로 최신화합니다:

- **요약 카드**: 총 매출, 진행중 업무 수, 이번 주 일정 수, 프로젝트 수 등의 핵심 지표 갱신
- **업무 목록**: `업무목록.csv` 기반으로 할 일 카드 갱신 (우선순위순 정렬, 상태 반영)
- **주간 일정**: `주간일정.txt` 기반으로 요일별 일정 갱신
- **프로젝트 진행률**: `프로젝트현황.csv` 기반으로 진행률 바 갱신
- **매출 차트**: `매출데이터.csv` 기반으로 차트 데이터 갱신
- **회의록/비고**: `회의록.txt`에서 주요 내용 반영

HTML을 직접 수정합니다. 기존 디자인(다크 테마, CSS 변수, card 클래스 등)을 그대로 유지하면서 데이터만 교체합니다.

### 3단계: 오늘의 브리핑

대시보드 업데이트 후, 사용자에게 오늘의 브리핑을 텍스트로 제공합니다.

브리핑 형식:

```
오늘의 브리핑 — [날짜]

[오늘 일정]
- 시간별 일정 목록

[긴급/중요 업무]
- 마감이 임박하거나 우선순위가 높은 업무

[프로젝트 현황 요약]
- 진행중인 프로젝트 상태 한 줄 요약

[주의사항/리마인드]
- 회의록에서 나온 액션 아이템
- 다가오는 마감일
```

브리핑은 **간결하고 실용적**으로 작성합니다. 불필요한 수식어 없이 핵심만 전달합니다.

## 주의사항

- dashboard.html의 기존 CSS 스타일과 구조를 유지합니다. 데이터만 교체합니다.
- CSV 파일의 인코딩(UTF-8)을 확인하고 한글이 깨지지 않도록 합니다.
- 오늘 날짜 기준으로 마감일이 지난 업무는 "지연" 표시를 합니다.
- 대시보드 업데이트가 끝나면 프리뷰 패널에서 확인 가능하다고 안내합니다.
